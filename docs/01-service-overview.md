# 01. 서비스 개요

## 무엇을 만들었는가

**Open Forms** — Google Forms 유사 설문 서비스. 폼 제작자가 설문을 만들고 공개 링크로 배포하면 누구나 로그인 없이 응답을 제출하고, 제작자는 대시보드에서 응답을 집계·시각화합니다.

### 핵심 사용자 흐름

```
[제작자] 회원가입/로그인
   → 폼 생성 (제목/설명)
   → 질문 추가 (객관식·다중선택·단답·장문·평점·숫자·날짜·시간·드롭다운 9종)
   → 발행 (DRAFT → PUBLISHED, 공개 slug 링크 생성)
   → 공개 링크 배포
[응답자] 공개 링크 접속 (로그인 불필요)
   → 응답 제출
[제작자] 대시보드
   → 총 응답 수 / 완료율 / 일별 응답 추이 / 질문별 응답 분포 확인
   → 폼 종료 (PUBLISHED → CLOSED, 추가 응답 차단)
```

## 왜 이 주제인가

기술테스트 평가 항목(RESTful API, 대시보드, 아키텍처, DB 설계, 인증/인가, 테스트)이 억지 없이 자연스럽게 전부 드러나는 도메인이기 때문입니다.

| 도메인 특성 | 드러나는 역량 |
|---|---|
| 제작자 인증 필요 / 응답자 익명 | 인증 API 구분, 401/403 처리 |
| 상태 전이 (DRAFT/PUBLISHED/CLOSED) | 상태 코드 다양성, 불변식 |
| 응답 집계 | 대시보드 집계·시각화 |
| 다층 구조 (폼-질문-선택지-응답-답변) | 1:N, 정규화, 인덱스 설계 |

## 구현 범위

- **필수**: RESTful API(폼/질문/응답 CRUD), React 대시보드
- **선택 ①**: JWT 기반 인증/인가 (리프레시 토큰 포함)
- **선택 ②**: 하이브리드 TDD (도메인 규칙은 실패 테스트 먼저)
- **비범위**: 실시간 협업, 파일 업로드, 이메일 알림

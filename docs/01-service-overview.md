# 01. 서비스 개요

## 무엇을 만들었는가

**Open Forms** 는 Google Forms 와 유사한 설문/폼 서비스입니다. 폼 제작자가 설문을 만들고 공개 링크로 배포하면, 누구나 로그인 없이 응답을 제출할 수 있고, 제작자는 대시보드에서 응답을 집계·시각화하여 확인합니다.

### 핵심 사용자 흐름

```
[제작자] 회원가입/로그인
   → 폼 생성 (제목/설명)
   → 질문 추가 (객관식·다중선택·단답·장문·평점 5종)
   → 발행 (DRAFT → PUBLISHED, 공개 slug 링크 생성)
   → 공개 링크 배포
[응답자] 공개 링크 접속 (로그인 불필요)
   → 응답 제출
[제작자] 대시보드
   → 총 응답 수 / 완료율 / 일별 응답 추이 / 질문별 응답 분포 확인
   → 폼 종료 (PUBLISHED → CLOSED, 추가 응답 차단)
```

## 왜 이 주제인가

기술테스트 평가 항목(RESTful API, 대시보드, 아키텍처, DB 설계, 문서화, 인증/인가, 테스트)이 **억지 없이 자연스럽게 전부 드러나는 도메인**이기 때문입니다. 
또한 짧은 일정의 기술테스트 기간에 무리없게 진행할 수 있는 프로젝트로 판단하였습니다.

| 도메인 특성 | 자연히 드러나는 평가 역량 |
|---|---|
| 제작자는 인증 필요, 응답자는 익명 | 인증 필요/불필요 API 구분, 401(미인증)/403(비소유자) 구분 |
| 폼 상태 전이 (DRAFT/PUBLISHED/CLOSED) | 상태 코드 다양성 (종료된 폼 응답 시 409 등) |
| 응답 데이터 집계 | "대시보드"의 요약·차트 시각화 (표 나열이 아닌 집계) |
| 폼–질문–선택지, 폼–응답–답변의 다층 구조 | 1:N / N:M 연관관계, 정규화, 인덱스 설계 |
| 응답 검증 (필수 질문, 선택지 유효성) | 유효성 실패 400, 일관된 에러 포맷 |

## 범위

### 구현 범위 (필수 + 선택 전부)
- **필수**: RESTful API(폼/질문/응답 CRUD), React 대시보드
- **선택 ①**: JWT 기반 인증/인가 (제작자만 자기 폼을 관리합니다)
- **선택 ②**: 테스트 코드 (하이브리드 TDD — 도메인 규칙은 테스트를 먼저 작성합니다)

### 명시적 비범위
- 응답자 계정/중복 응답 방지(이메일 인증 등)
- 실시간 협업 편집, 파일 업로드 질문 타입
- 이메일 알림, 폼 템플릿
